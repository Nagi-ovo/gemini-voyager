# 用量狀態列

Gemini 2026 替對話加上了用量額度,但要看還剩多少,得專門點進 `gemini.google.com/usage` 整頁裡翻。

Voyager 把「每日」和「每週」兩個額度做成一個**可拖曳的小浮條**,常駐在聊天介面,隨時瞄一眼,不必離開對話。

![用量狀態列](/assets/gemini-usage-status.png)

## 它長什麼樣

一個精簡的迷你條:方案徽章(如 `PRO`)+ 兩條細進度條(每日 / 每週)+ 百分比,外加一個重新整理按鈕和一個打開原生用量頁的小圖示。半透明、低調,不搶正文。

## 它是怎麼運作的

- **可拖曳 + 記住位置**:在條上任意位置按住拖動,放到你順手的地方;鬆手即記住,跨重新整理、跨導覽、跨分頁都保留。預設在輸入框上方置中。
- **背景靜默更新**:資料由擴充功能在背景自動更新——**你不用重新整理網頁、也不用點進 `/usage`**。每次回覆結束幾秒後會自動更新一次(這時額度剛好變了);閒置時也會隔幾分鐘做一次保守的兜底更新。
- **滑鼠停留看細節**:把滑鼠停在某一條上會顯示該額度的重置時間;停在整條上顯示「剛剛更新 / X 分鐘前更新」。
- **兩個各司其職的按鈕**:
  - **重新整理 ↻** — 立刻在背景靜默更新一次(原地轉一下、更新數字,**絕不跳轉**)。
  - **開啟 ↗** — 在新分頁開啟原生 `/usage` 頁面。這是整個條上**唯一**會跳轉的地方。

## 怎麼用

1. 開啟 Voyager 設定面板(瀏覽器工具列上的擴充功能圖示)。
2. 打開 **用量狀態列** 開關(預設關閉)。
3. 小浮條會立刻出現在聊天介面,拖到你想要的位置即可。

::: tip 開箱即用
啟用後 Voyager 會在背景自動取一次用量,**不需要你先點進 `/usage`**。如果哪天 Google 改了內部介面導致取不到,造訪一次 `gemini.google.com/usage`,Voyager 會就地對照頁面上的真實數字重新校準。
:::

## 更新頻率與偵測

更新是**事件驅動**的:只在用量真正變化(你發了訊息)之後才重新整理,外加一個保守的閒置兜底,**不做高頻輪詢**。每次更新就是網頁本身用來取用量的那個請求、用的是你本人的登入狀態,頻率和正常使用沒有區別——請求量大致是「每輪對話一次」,對 Google 偵測的影響極低。

## 關於隱私

- 用量數值與浮條位置都**只存在本機** (`chrome.storage.local`),不上傳任何伺服器。
- 不讀取也不快取任何對話內容——只取兩個額度百分比、重置時間和方案名稱。
- 關掉開關,小浮條即移除;快取仍留在本機,再啟用時無需重新載入。

## 平台

僅支援 **Google Gemini**(`gemini.google.com`)。
