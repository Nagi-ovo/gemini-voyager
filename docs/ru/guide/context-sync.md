# Перенос памяти: Синхронизация контекста

**Разные измерения, плавный обмен**

Разрабатывайте логику в веб-интерфейсе и внедряйте код в IDE. Gemini Voyager разрушает барьеры между измерениями, мгновенно наделяя вашу IDE «процессом мышления» из веб-версии.

## Забудьте о постоянных переключениях

Самое неприятное для разработчика: подробно обсудить решение в вебе, а вернувшись в VS Code/Trae/Cursor, объяснять требования заново, как незнакомцу. Из-за лимитов и скорости отклика веб — это «мозг», а IDE — «руки». Voyager позволяет им делить одну душу на двоих.

## Три простых шага к синхронности

1. **Пробудите CoBridge**: Установите плагин **CoBridge** из VS Code Marketplace и запустите его. Это мост между веб-страницей и локальной машиной.
   ![Расширение CoBridge](/assets/CoBridge-extension.png)

   ![Сервер CoBridge включен](/assets/CoBridge-on.png)

2. **Рукопожатие**:
   - Включите «Синхронизацию контекста» в настройках Voyager.
   - Совместите номера портов. Надпись «IDE Online» означает, что соединение установлено.

   ![Панель синхронизации контекста](/assets/context-sync-console.png)

3. **Синхронизация в один клик**: Нажмите **"Sync to IDE"**.

   ![Синхронизация завершена](/assets/sync-done.png)

## Укоренение

После завершения синхронизации в корневом каталоге вашей IDE появится файл `.vscode/AI_CONTEXT_SYNC.md`. Будь то Trae, Cursor или Copilot, они автоматически прочитают эту «память» через свои соответствующие файлы правил.

## Принципы

- **Чистота**: CoBridge автоматически правит `.gitignore`, чтобы ваши приватные диалоги не попали в Git-репозиторий.
- **Профессионализм**: Полный формат Markdown, благодаря чему ИИ в IDE читает контекст так же легко, как инструкцию.
- **Совет**: Если диалог был давно, сначала прокрутите его вверх с помощью [Таймлайна], чтобы веб-версия «вспомнила» контекст — тогда результат синхронизации будет лучше.

- **[Посетить репозиторий на GitHub](https://github.com/Winddfall/CoBridge)**: Узнайте больше о логике CoBridge или поддержите проект «синхронизации душ» звездой.

> **Большие модели больше не теряют память — готовы к работе сразу.**
