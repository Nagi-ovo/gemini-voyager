# Перенос расширения Safari

::: warning Нужно один раз выполнить вручную
Начиная с **v1.6.0**, хост-приложение Safari переименовано с «**Gemini Voyager**» на «**Voyager**». macOS определяет приложения по имени, поэтому если просто установить новую версию, она останется рядом со старым приложением, что может привести к дублированию расширения или сбоям в работе. Выполните эту замену один раз — после этого автоматические обновления продолжатся как обычно.
:::

## Ваши данные в сохранности

Bundle ID приложения не изменился. Ваши папки, библиотека промптов, облачная синхронизация и все настройки сохраняются. Этот шаг лишь заменяет само приложение и не затрагивает данные.

## Шаги переноса

1. **Полностью завершите Safari** (нажмите `⌘Q` в Safari, а не просто закройте окно).
2. Откройте **Finder → Программы** и перетащите старое «**Gemini Voyager.app**» в Корзину.
3. Откройте только что загруженный DMG и перетащите «**Voyager.app**» в **Программы**.
4. Снова откройте Safari → **Настройки → Расширения** и включите «**Voyager Extension**».

## Два «не»

- ❌ **Не оставляйте оба приложения.** Если оставить старое «Gemini Voyager.app», два расширения будут конфликтовать.
- ❌ **Не нажимайте «Удалить» для старого расширения на вкладке «Расширения» в Safari.** Это ссылается на старое приложение и только всё запутает. Просто перетащите старое приложение в Корзину, как в шаге 2.

## После этого

После одной замены будущие версии Safari обновляют новый «Voyager» через встроенный автообновлятор (Sparkle) — вручную менять больше не придётся.
