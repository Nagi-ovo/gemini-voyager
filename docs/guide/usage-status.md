# 用量状态条

Gemini 2026 给对话加上了用量额度，但要看还剩多少，得专门点进 `gemini.google.com/usage` 整页里翻。

Voyager 把「每日」和「每周」两个额度做成一个**可拖拽的小浮条**，常驻在聊天界面，随时瞄一眼，不用离开对话。

![用量状态条](/assets/gemini-usage-status.png)

## 它长什么样

一个紧凑的迷你条：套餐徽章（如 `PRO`）+ 两条细进度条（每日 / 每周）+ 百分比，外加一个刷新按钮和一个打开原生用量页的小图标。半透明、低调，不抢正文。

## 它是怎么工作的

- **可拖拽 + 记住位置**：在条上任意位置按住拖动，放到你顺手的地方；松手即记住，跨刷新、跨导航、跨标签页都保留。默认在输入框上方居中。
- **后台静默更新**：数据由插件在后台自动刷新，**你不用刷新网页、也不用点进 `/usage`**。每次发完一条消息、回复结束后会自动更新一次（这时额度刚好变了）；闲置时也会隔几分钟兜底刷新一次。
- **悬停看细节**：鼠标停在某一条上会显示该额度的重置时间；停在整条上显示「刚刚更新 / X 分钟前更新」。
- **两个按钮各司其职**：
  - **刷新 ↻** — 立刻在后台静默更新一次（原地转一下、更新数字，**不跳转**）。
  - **打开 ↗** — 在新标签页打开原生 `/usage` 页面。这是整个条上**唯一**会跳转的地方。

## 怎么用

1. 打开 Voyager 设置面板（浏览器右上角插件图标）。
2. 打开 **用量状态栏** 开关（默认关闭）。
3. 小浮条会立刻出现在聊天界面，拖到你想要的位置即可。

::: tip 开箱即用
启用后插件会在后台自动取一次用量，**不需要你先点进 `/usage`**。如果哪天 Google 改了内部接口导致取不到，访问一次 `gemini.google.com/usage`，Voyager 会就地对照页面上的真实数字重新校准。
:::

## 更新频率与检测

更新是**事件驱动**的：只在用量真正变化（你发了消息）之后才刷新，外加一个保守的闲置兜底，**不做高频轮询**。每次刷新就是网页本身用来取用量的那个请求、用的是你本人的登录态，频率和正常使用没有区别——请求量大致是「每轮对话一次」，对 Google 的检测影响极低。

## 关于隐私

- 用量数值与浮条位置都**只存在本地** (`chrome.storage.local`)，不上传任何服务器。
- 不读取也不缓存任何对话内容——只取两个额度百分比、重置时间和套餐名。
- 关掉开关，小浮条即移除；缓存仍留在本地，再开启无需重新加载。

## 平台

仅支持 **Google Gemini**（`gemini.google.com`）。
