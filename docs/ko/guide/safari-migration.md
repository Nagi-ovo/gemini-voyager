# Safari 확장 프로그램 마이그레이션 안내

::: warning 한 번의 수동 작업이 필요합니다
**v1.6.0**부터 Safari 호스트 앱 이름이 "**Gemini Voyager**"에서 "**Voyager**"로 변경되었습니다. macOS는 앱을 이름으로 인식하므로, 새 버전을 그대로 설치하면 기존 앱과 함께 남아 확장 프로그램이 중복되거나 동작이 꼬일 수 있습니다. 아래 단계로 한 번만 교체하면 이후 자동 업데이트는 평소대로 진행됩니다.
:::

## 데이터는 사라지지 않습니다

앱의 Bundle ID는 바뀌지 않았습니다. 폴더, 프롬프트 라이브러리, 클라우드 동기화, 모든 설정이 그대로 유지됩니다. 이 단계는 앱 본체만 교체하며 데이터는 건드리지 않습니다.

## 마이그레이션 단계

1. **Safari를 완전히 종료합니다**(Safari에서 `⌘Q`. 창만 닫는 것으로는 부족합니다).
2. "**Finder → 응용 프로그램**"을 열고 기존 "**Gemini Voyager.app**"을 휴지통으로 드래그합니다.
3. 새로 내려받은 DMG를 열고 안에 있는 "**Voyager.app**"을 "**응용 프로그램**"으로 드래그합니다.
4. Safari를 다시 열어 "**설정 → 확장 프로그램**"에서 "**Voyager Extension**"을 활성화합니다.

## 하지 말아야 할 두 가지

- ❌ **두 앱을 동시에 남겨두지 마세요.** 기존 "Gemini Voyager.app"을 그대로 두면 두 확장 프로그램이 충돌합니다.
- ❌ **Safari의 "확장 프로그램" 화면에서 기존 확장의 "제거"를 누르지 마세요.** 그것은 기존 앱을 가리켜 오히려 더 혼란스러워집니다. 2단계처럼 기존 앱을 휴지통으로 드래그하세요.

## 이후

한 번만 교체하면 이후 Safari 버전은 내장 자동 업데이트(Sparkle)를 통해 새 "Voyager"로 업데이트됩니다. 더 이상 수동 교체가 필요 없습니다.

궁금한 점은 [GitHub Issues](https://github.com/Nagi-ovo/voyager/issues)로 알려주세요.
