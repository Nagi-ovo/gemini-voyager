# 사이드바의 최근 Gems

Gemini 2026 리디자인은 처음에 Gems를 설정 메뉴 뒤로 옮겼다가, 조용히 사이드바에 nav 항목을 다시 넣었습니다 — 하지만 단지 `/gems/view`로 튕겨내는 링크일 뿐입니다.

Voyager는 네이티브 Gems 항목이 사이드바 안에서 최근 gems 목록으로 "펼쳐지도록" 만듭니다.

## 어떻게 보이나요

- **네이티브 Gems 항목 아래에 매달려 있습니다.** Gemini 자체의 "Gems" 레이블에 맞춰 들여쓰기되어, 붙여 넣은 패널이 아니라 그 항목의 서브리스트로 읽힙니다.
- **셰브론 토글.** Gems 항목의 오른쪽에 있는 작은 `›` 버튼이 열리면 `⌄`로 회전합니다. 클릭하여 접기/펼치기. 상태는 `chrome.storage.local`에 저장되고 탭 간 동기화됩니다.
- **네트워크 트래픽 없음.** 목록은 마지막으로 `https://gemini.google.com/gems/view`를 방문했을 때 채워진 로컬 캐시에서 읽힙니다. API 호출도, 폴링도, 백그라운드 페치도 없습니다.

## 사용법

1. Voyager 팝업을 엽니다 (툴바의 확장 프로그램 아이콘).
2. **사이드바의 최근 Gems** 슬라이더를 찾습니다.
3. 원하는 수(1–10)로 드래그합니다. **`0`은 섹션을 완전히 숨깁니다** — 기능이 필요 없으면 거기에 두세요.

::: tip 첫 사용 시
활성화 후 gems가 하나도 보이지 않으면 로컬 캐시가 비어 있다는 뜻입니다. `gemini.google.com/gems/view`를 한 번 방문하세요 — Voyager가 조용히 gems 목록을 스냅샷합니다. 다음에 어떤 Gemini 페이지에 있든 목록이 거기에 있을 것입니다.
:::

## 캐시가 갱신되는 시점

Voyager는 **`/gems/view`에 있을 때만** 캐시를 갱신합니다:

- 페이지 방문, 재정렬, 이름 변경, 생성, 삭제 — 모두 실시간으로 캐시에 동기화됩니다.
- `/gems/view` 밖에서는 스크래핑이 일어나지 않습니다.

다른 기기에서 gem을 추가하면, Voyager가 "마법처럼" 알지는 못합니다. 이 컴퓨터에서 `/gems/view`를 한 번 열면 동기화됩니다.

## 개인정보

- 데이터는 **로컬 브라우저 저장소** (`chrome.storage.local`)에 머뭅니다. 어디에도 업로드되지 않습니다.
- gem의 대화 내용은 읽지도 캐시하지도 않습니다 — 이름, 설명, 링크, 아바타의 첫 글자만입니다.
- 기능 비활성화(개수 = 0)는 캐시를 그대로 두므로 재활성화는 즉시 반영됩니다.

## 플랫폼

Gemini만 (`gemini.google.com`). AI Studio의 gem 항목은 형태가 다르며 적용되지 않습니다.
