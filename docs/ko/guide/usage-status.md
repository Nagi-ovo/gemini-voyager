# 사용량 상태 바

Gemini 2026은 대화에 사용량 한도를 추가했지만, 얼마나 남았는지 보려면 `gemini.google.com/usage` 전체 페이지로 이동해야 합니다.

Voyager는 **일일** 및 **주간** 한도를 채팅 UI 안에 그대로 머무는 작은 **드래그 가능한 플로팅 바**로 바꿔 줍니다 — 대화를 벗어나지 않고 언제든 슬쩍 확인하세요.

![사용량 상태 바](/assets/gemini-usage-status.png)

## 어떻게 보이나요

작고 단정한 미니 바: 요금제 배지(예: `PRO`), 백분율이 표시된 두 개의 가는 진행 바(일일 / 주간), 새로고침 버튼, 그리고 네이티브 사용량 페이지를 여는 작은 아이콘. 반투명하고 차분해서 대화를 방해하지 않습니다.

## 어떻게 작동하나요

- **드래그 가능 + 위치 기억**: 바의 아무 곳이나 잡아 원하는 곳에 놓으세요. 위치는 새로고침, 페이지 이동, 탭 전환을 거쳐도 유지됩니다. 기본값은 입력창 바로 위 가운데입니다.
- **백그라운드에서 조용히 새로고침**: 데이터는 알아서 갱신됩니다 — **페이지를 다시 불러오거나 `/usage`를 열 필요가 전혀 없습니다**. 각 응답이 끝나고 몇 초 뒤(사용량이 막 바뀌는 바로 그 시점)에 갱신되며, 유휴 상태에서는 몇 분마다 보수적인 백업 갱신이 한 번씩 돕니다.
- **호버로 세부 정보 확인**: 바 하나에 마우스를 올리면 해당 한도의 리셋 시각이 보이고, 바 전체에 올리면 "방금 업데이트됨 / X분 전 업데이트됨"이 표시됩니다.
- **목적이 분명한 두 개의 컨트롤**:
  - **새로고침 ↻** — 즉시 조용히 갱신을 강제합니다(제자리에서 빙글 돌며 갱신될 뿐 **절대 이동하지 않습니다**).
  - **열기 ↗** — 네이티브 `/usage` 페이지를 새 탭에서 엽니다. 바에서 이동하는 것은 이것 **하나뿐**입니다.

## 사용법

1. Voyager 설정 패널을 엽니다(브라우저 도구 모음의 확장 프로그램 아이콘).
2. **사용량 상태 바** 토글을 켭니다(기본값은 꺼짐).
3. 플로팅 바가 채팅 UI에 즉시 나타납니다 — 원하는 곳으로 드래그하세요.

::: tip 설치하면 바로 작동
활성화하면 Voyager가 백그라운드에서 사용량을 자동으로 가져옵니다 — **먼저 `/usage`를 방문할 필요가 없습니다**. 만약 Google이 내부 API를 바꿔 숫자가 더 이상 들어오지 않으면, `gemini.google.com/usage`를 한 번만 열어 주세요. Voyager가 그 페이지에 실제로 표시된 값을 기준으로 다시 보정합니다.
:::

## 갱신 빈도 및 탐지

갱신은 **이벤트 기반**입니다: 바는 사용량이 실제로 바뀐 뒤(즉, 메시지를 보낸 뒤)에만 새로고침되며, 여기에 보수적인 유휴 백업이 더해질 뿐 — **잦은 폴링은 하지 않습니다**. 각 갱신은 페이지 자체가 사용량을 가져올 때 쓰는 바로 그 요청을, 당신의 로그인 세션으로, 사람 같은 빈도로 보내는 것입니다. 요청량은 대략 "대화 한 턴당 한 번" 수준이라, Google의 탐지에 미치는 영향은 무시할 만합니다.

## 개인정보

- 사용량 숫자와 바의 위치는 모두 **로컬에만** 저장됩니다(`chrome.storage.local`) — 어떤 서버에도 업로드되지 않습니다.
- 대화 내용은 읽지도 캐시하지도 않습니다 — 두 개의 백분율, 리셋 시각, 요금제 이름만 가져옵니다.
- 토글을 끄면 바가 제거됩니다. 캐시는 로컬에 그대로 남으므로 다시 켤 때 새로고침이 필요 없습니다.

## 플랫폼

**Google Gemini**(`gemini.google.com`)만 지원합니다.
